Just like in Java, the add method of the List interface returns a boolean, indicating whether the Collection has been changed by the add operation - which always is true for Lists. One might have guessed that Kotlin instead returns a new List with the element appended, which is not the case. Due to type inference this is not obvious. If you would rather append an element to a list, you can write this val listB = listA + 4, making use of the extension + operator of Collection.
